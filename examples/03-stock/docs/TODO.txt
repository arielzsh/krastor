## 一、Business Workflow (4 Phases)

### Phase 1: Asset Initialization (Genesis)

- [ ] Deploy Token 2022 Mint with multiple extensions:
  - [ ] `Transfer Hook` extension
  - [ ] `Metadata Pointer` extension
- [ ] Write real-world stock metadata on-chain:
  - [ ] AAPL CUSIP/ISIN code
  - [ ] Legal total shares outstanding
  - [ ] Custodian bank vault address
- [ ] Initialize global config PDA:
  - [ ] `split_multiplier: u64 = 1`
  - [ ] `market_status: enum { OPEN, CLOSED, PRE_MARKET }`
  - [ ] `base_stamp_tax_bps: u64 = 10` (0.1%)

### Phase 2: Daily Trading & Risk Control (Transfer Hook Lifecycle)

- [ ] Implement `execute_transfer_hook` (Token 2022 CPI):
  - [ ] **Market hours check**: reject transfers when `market_status == CLOSED`
    - [ ] Error code: `0x1702 (StockMarketClosed)`
  - [ ] **Velocity tracker**: read `Velocity` account, compute transfer frequency
    - [ ] Dynamic stamp tax: `0.1% → 5.0%` based on velocity
    - [ ] Smooth ramp formula: `tax_bps = base + velocity * scaling_factor`
  - [ ] **Atomic fund split**: 95% to recipient, 5% stamp tax → burn account
- [ ] Velocity tracking PDA:
  - [ ] `window_start: i64` (unix timestamp)
  - [ ] `transfer_count: u64` (within window)
  - [ ] `window_duration: i64` (e.g., 60 seconds)

### Phase 3: Corporate Actions — Stock Split

- [ ] Implement `trigger_stock_split(ratio: u64)`:
  - [ ] Admin-only (check `ctx.accounts.authority.key() == global_config.admin`)
  - [ ] Update `global_config.split_multiplier` from 1 to `ratio`
  - [ ] **Key demo**: user token balances remain UNCHANGED on-chain
  - [ ] View function `get_effective_balance(user)` does `balance * split_multiplier`
- [ ] Demonstrate: 1:5 split → frontend shows 5× balance, chain data unmodified

### Phase 4: Dividend Distribution (Snapshot + Claim)

- [ ] Implement `snapshot_dividend(dividend_per_share: u64)`:
  - [ ] Lock block height / slot number
  - [ ] Record all holder "physical balances" (pre-split-multiplier)
  - [ ] Fund dividend pool PDA with USDC (Simulated)
- [ ] Implement `claim_dividend(user)`:
  - [ ] Fixed-point math: `user_share = user_balance * 1_000_000 / total_supply`
  - [ ] `usdc_payout = user_share * dividend_per_share / 1_000_000`
  - [ ] Release USDC from dividend pool PDA
  - [ ] Mark user as "claimed" to prevent double-claim

## 四、Intentional Bugs (Krastor Fuzzer Targets)

| # | Bug | Location | Proptest | Krastor |
|---|-----|----------|----------|---------|
| 1 | Split multiplier overflow in view fn | `get_effective_balance()`: `balance * multiplier` | ~0% — random rarely hits overflow | `flip_data` 40%/round |
| 2 | Velocity counter window race | `execute_transfer_hook()`: concurrent writes to `Velocity` | 0% — no parallel model | auto-sequence + `flip_data` |
| 3 | Dividend double-claim | `claim_dividend()`: no claimed flag check | 0% — no multi-claim model | auto-sequence: claim → claim |
| 4 | Market hours bypass via Clock sysvar | `execute_transfer_hook()`: `Clock::unix_timestamp` | 0% — no sysvar model | `flip_data` targets Clock |
| 5 | Dividend pool USDC not fully funded | `claim_dividend()`: pool balance < total claims | 0% — no pool invariant | `zero_lamports` drains pool |
| 6 | Metadata pointer overwrite | `initialize()`: no existence check | 0% — no re-init model | auto-sequence: init → init |
| 7 | Stamp tax calculation overflow | `execute_transfer_hook()`: `amount * tax_bps / 10000` | ~0% — rare extreme | `flip_data` 40%/round |
| 8 | Split effective date manipulation | `trigger_stock_split()`: future date not validated | 0% — no time model | `flip_data` + Clock |

## 六、Account Layout (Solana)

```
GlobalConfig (PDA, seeds=[b"config"]):
  admin: Pubkey            // 32 bytes
  split_multiplier: u64    // 8 bytes
  market_status: u8        // 1 byte (0=OPEN, 1=CLOSED, 2=PRE)
  base_stamp_tax_bps: u64  // 8 bytes
  bump: u8                 // 1 byte

Velocity (PDA, seeds=[b"velocity", user.key()]):
  owner: Pubkey            // 32 bytes
  window_start: i64        // 8 bytes
  transfer_count: u64      // 8 bytes
  bump: u8                 // 1 byte

DividendSnapshot (PDA, seeds=[b"dividend", slot]):
  slot: u64                // 8 bytes
  total_supply: u64        // 8 bytes (physical snapshot)
  dividend_per_share: u64  // 8 bytes (in USDC smallest unit)
  pool_lamports: u64       // 8 bytes

DividendClaim (PDA, seeds=[b"claim", user.key(), slot]):
  claimed: bool            // 1 byte
  amount: u64              // 8 bytes
```

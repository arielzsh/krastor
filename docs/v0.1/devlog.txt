## 项目概述

Krastor 是 Ackee Blockchain 的 Trident 框架的再生，一个面向 Solana 程序的覆盖率引导 Fuzz。

### 核心架构

```
krastor/
├── crates/
│   ├── fuzz-core/        ✅ 核心引擎：随机序列 + 账户突变 + LiteSVM + 不变量
│   ├── idl-parser/       ✅ Anchor IDL JSON 解析 + harness 代码生成
│   ├── instrumentor/     ✅ SBF 字节码静态插桩 + 覆盖位图（可选模块）
│   ├── cli/              ✅ cargo-krastor CLI
│   └── report/           ✅ 覆盖率 HTML 报告 + crash JSON 持久化
└── examples/
    └── vulnerable/       ✅ 含 3 个已知漏洞的 Anchor 合约
```

### 三个核心优势

1. **覆盖率引导** — 不是随机撞大运，是系统性逼近未覆盖分支
2. **跨指令序列编排** — 自动尝试任意长度、顺序的组合（闪电贷模式）
3. **Solana 账户感知** — owner 替换、lamports 归零、data 清空等定向攻击

### 三个劣势与对策

| 劣势 | 对策 |
|------|------|
| SBF 插桩脆弱 | 插桩层独立 crate，降级为可选模块 |
| 外部依赖长 | 只用 LiteSVM（零外部进程） |
| 缺少精简 | 内置二分+贪心 Shrinking |
